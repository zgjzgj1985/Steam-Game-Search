## 一、核心目标（第一性原理）

### 1.1 根本目的

**从海量游戏中发现独特、令人着迷的游戏设计**

不是找共性、不是找趋势，而是发现那些让人眼前一亮、印象深刻、独特的创新点。

### 1.2 什么标签真正有价值？

| 类型 | 定义 | 示例 | 发现价值 |
|------|------|------|----------|
| **品类标配** | 每款游戏都有 | RPG、回合制、卡牌 | 无 |
| **机制层** | 某类游戏的共性 | 肉鸽、牌组构建 | 筛选用 |
| **独特亮点** | 单款游戏的招牌 | 形态融合、骰子驱动 | **发现价值** |

**真正有价值的标签必须满足：**
1. **稀有性**：只出现在少数游戏中（1-5款）
2. **吸引力**：看到这个标签就想玩
3. **独特性**：是这款游戏的标志性卖点
4. **可描述**：能让人理解这是什么机制

## 二、标签分类体系

### 2.1 三层标签架构

```
┌─────────────────────────────────────────┐
│  第一层：品类标配（已过滤）              │
│  RPG / JRPG / 回合制 / 卡牌 等          │
├─────────────────────────────────────────┤
│  第二层：机制层（用于筛选）              │
│  肉鸽融合 / 牌组构建 / 形态融合 等      │
├─────────────────────────────────────────┤
│  第三层：独特亮点（用于发现）            │
│  骰子驱动 / 类魂机制 / 契约驯服 等      │
└─────────────────────────────────────────┘
```

### 2.2 各层标签用途

| 层级 | 用途 | 筛选逻辑 |
|------|------|----------|
| 品类标配 | 过滤 | 全部加入黑名单 |
| 机制层 | 分类/筛选 | 按出现频率统计，用于筛选面板 |
| 独特亮点 | **发现** | 按独特程度排序，用于展示卡片 |

## 三、标签质量评估标准

### 3.1 评价维度

| 维度 | 定义 | 权重 |
|------|------|------|
| **独特性** | 只出现在少数游戏中 | 40% |
| **吸引力** | 让人看了想玩 | 30% |
| **可学习性** | 能提取为设计参考 | 20% |
| **描述清晰度** | 一看就懂是什么机制 | 10% |

### 3.2 评分公式

```
创新价值 = 独特性×0.4 + 吸引力×0.3 + 可学习性×0.2 + 清晰度×0.1
```

**独特性评分**（出现次数越少分数越高）：
- 出现1次：100分
- 出现2次：80分
- 出现3次：60分
- 出现4-5次：40分
- 出现6次以上：20分（接近机制层）

### 3.3 展示优先级

```
按创新价值降序展示
  ↓
Top 标签 = 高独特性 + 高吸引力 + 高可学习性
```

## 五、标签生成流程

### 5.1 当前流程

```
1. LLM分析游戏 → 生成 rawMechanics（原始标签）
2. 聚类映射 → 将相似标签归入机制层
3. 过滤黑名单 → 移除品类标配
4. 展示时按频率排序（❌ 错误）
```

### 5.2 问题分析

**当前系统的核心问题：按频率排序导致展示的都是品类标配**

- 高频标签 = 品类共性 = 没有"发现"价值
- 低频标签 = 独特创新 = **才是值得学习的**

**示例对比**：

| 标签 | 出现次数 | 按频率排序 | 按独特性排序 |
|------|----------|------------|--------------|
| 肉鸽融合 | 27次 | 第1名 ❌ | 被淹没 |
| 形态融合 | 7次 | 第5名 ❌ | 第2名 ✅ |
| 骰子驱动 | 2次 | 第50名 ❌ | 第1名 ✅ |

### 5.3 改进后的流程

```
1. LLM分析游戏 → 生成 rawMechanics（原始标签）
2. 聚类映射 → 将相似标签归入机制层（保留rawMechanics原始语义）
3. 过滤黑名单 → 移除品类标配
4. **按创新价值排序** → 独特性优先
5. **只展示亮点标签** → 过滤机制层标签
```

## 五、LLM标签采集Prompt问题诊断

### 5.4 当前Prompt与新思路的根本冲突

**涉及文件**: `scripts/batch_generate_llm_tags.py`

**当前Prompt关键问题**:

```
【标签要求】
...
- 优先使用已有的高频标签   ← ❌ 与新思路冲突！
```

**问题分析**:

| 方向 | 当前Prompt | 新思路 |
|------|-----------|--------|
| 标签选择 | 优先高频标签 | **优先独特标签** |
| 标签类型 | 机制层标签为主 | **独特亮点标签为主** |
| 生成数量 | 3-6个 | 2-4个（精选） |
| 评判标准 | 高频=通用 | **低频=独特=有价值** |

### 5.5 当前Prompt全文

```
你是创新玩法标签分析师。请根据游戏信息生成精准的创新玩法标签。

【任务】
分析游戏的 Steam 标签、类型、描述，识别其核心创新玩法，生成 3-6 个精准的玩法标签。

【标签要求】
- 必须是具体的机制标签（如：肉鸽融合、牌组构建、形态融合、随机地牢）
- 不能是泛化标签（如：冒险、动作、RPG）
- 不能是美术风格标签（如：像素、2D、卡通）
- 优先使用已有的高频标签  ← 问题所在！

【参考标签库】
肉鸽融合, 牌组构建, 形态融合, 随机地牢, 开放世界, 基地建设...
```

### 5.6 问题根因

1. **"优先使用已有的高频标签"** → 鼓励LLM生成机制层标签
2. **参考标签库** → 都是机制层标签，缺乏独特亮点示例
3. **生成数量3-6个** → 追求数量，忽视质量

### 5.7 改进后的Prompt（建议）

```
你是创新玩法标签分析师。请从游戏信息中挖掘真正独特、令人着迷的设计亮点。

【任务】
识别这款游戏最独特、最吸引人的设计亮点，生成 2-4 个精准的创新标签。

【标签要求】
- **必须是这款游戏的独特卖点**（不是所有游戏都有的共性）
- **让人看到这个标签就想玩**
- **能提取为设计参考**
- **不能是泛化品类词**（如：冒险、动作、RPG、回合制）

【标签类型优先级】
1. 独特亮点（只出现在少数游戏中） ← 最重要
2. 创新组合（玩法A+玩法B的独特融合）
3. 差异化机制（非传统的实现方式）

【输出格式】
只输出合法的 JSON：
{
  "tags": ["标签1", "标签2", "标签3"],
  "summary": "一句话总结游戏的独特亮点（20-50字）"
}
```

### 5.8 预期效果对比

| 指标 | 当前Prompt | 改进后 |
|------|-----------|--------|
| 标签类型 | 机制层为主 | 独特亮点为主 |
| 出现频率 | 期望高频 | 期望低频 |
| 吸引力 | 一般 | **让人想玩** |
| 独特性 | 较弱 | **较强** |

## 十、总结

### 核心原则

1. **发现价值 > 共性统计**：找独特亮点，而不是找共同趋势
2. **低频 ≠ 低价值**：恰恰相反，低频标签往往是最独特的
3. **按独特性排序**：让用户第一眼看到的是让人着迷的标签

### 验收标准

- [ ] 卡片上展示的标签都是"令人想玩"的亮点标签
- [ ] 品类标配和机制层标签不会出现在卡片上
- [ ] 独特标签（出现1-5次）排在前面
- [ ] 每个独特标签都能让人理解是什么机制
