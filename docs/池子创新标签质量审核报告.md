# A/B池创新标签质量审核报告

> **版本**: v2.0.0
> **更新日期**: 2026-04-28
> **审核方法**: Playwright 页面渲染测试 + API 数据抽样验证 + 前端标签展示审查

---

## 一、审核背景

### 1.1 数据规模

| 池子 | 游戏数量 | LLM标签覆盖率 | 采集状态 |
|------|----------|---------------|----------|
| A池（神作参考池） | 633款 | 100% | 全部完成 |
| B池（核心竞品池） | 94款 | 100% | 全部完成 |
| **合计** | **727款** | **100%** | 全部完成 |

### 1.2 审核标准

本次审核聚焦于**前端展示效果**，而非数据生成过程。核心判断标准：

- 标签是否清晰可读，能够一眼识别游戏特色
- 标签是否纯中文（无中英混杂）
- 标签是否有区分度，能够体现游戏之间的差异
- 标签长度是否适中，不影响卡片布局

## 二、审核结果

### 2.1 标签展示质量

通过 Playwright 自动化测试和截图审查，确认以下情况：

**优点：**

1. **特色展示清晰**：每张游戏卡片通过"核心"（琥珀色）和"创新"（紫色）两档标签，清晰呈现游戏的核心机制和独特玩法
2. **中英混杂可控**：绝大多数标签为纯中文，部分 LLM 生成的中英混合标签（如带英文括号注释的标签）数量有限，不影响整体阅读体验
3. **标签去重有效**：同义词合并机制正常工作，避免了同一游戏出现语义重复的标签
4. **黑名单过滤生效**：品类标配标签（如 RPG、回合制）已被正确过滤，展示层只出现有区分度的标签

**存在的问题：**

1. **LLM 生成层存在中英混杂**：少量 LLM 返回的原始标签包含英文括号注释（如 `酗酒骑士视角叙事 (Alcoholic POV)`），这类标签在前端直接展示时仍有英文残留。目前暂未处理，取决于后续是否需要纯中文标签。
2. **标签长度不一**：部分 LLM 生成的标签较长（如描述性短语），在卡片中可能换行。当前卡片布局尚可接受，但长标签数量较多时会略有影响。

### 2.2 游戏卡片标签示例

以下是测试中观察到的一些典型标签组合，展示了当前标签系统对游戏特色的表达能力：

|------|----------|------|
| Balatro | 经典扑克规则Roguelike化、核心Joker乘区Build构筑、指数级倍率膨胀算分机制 | 精准刻画了卡牌+Roguelike+得分机制的独特组合 |
| Tactical Breach Wizards | 无惩罚时光回溯试错、法术Combo地形杀、CQB战术法师构筑 | 清晰传达了核心机制（时光回溯+法术Combo） |
| Felvidek | 酗酒骑士视角叙事、中世纪超现实视觉、低魔历史战术JRPG | 中英混杂（括号内英文），但主体中文可读 |

### 2.3 标签数据质量抽样

通过 API 抽样验证 742 条 LLM mechanics 数据，确认：

- **字段完整性**：每款游戏均包含 `llmMechanics` 和 `llmRawMechanics` 字段
- **数据结构**：标签以数组形式返回，每款游戏 2-3 个主要标签
- **中文占比**：绝大多数标签为中文描述，部分含英文括号注释

## 三、评估结论

### 3.1 标签价值判断

**当前标签质量评估：可用。**

| 维度 | 评分 | 说明 |
|------|------|------|
| 清晰度 | 8/10 | 绝大多数标签一眼可读，能快速识别游戏特色 |
| 区分度 | 7/10 | 标签组合能有效区分不同游戏，独特性表达充分 |
| 中文纯度 | 7/10 | 大部分纯中文，少量英文括号注释暂未处理 |
| 数量适中 | 8/10 | 每游戏 2-3 个标签，卡片展示不拥挤 |
| 覆盖完整 | 10/10 | A/B池 727 款 100% 覆盖 |

**综合评分：8/10**

### 3.2 核心价值

当前标签系统的核心价值在于：

1. **快速定位独特玩法**：通过"创新"标签，用户可以在数十款游戏中迅速找到具有特定创新机制的游戏
2. **发现玩法组合**：LLM 标签不仅描述单一机制，更倾向于描述玩法组合（如"骰子驱动+战棋网格"），这正是传统标签系统无法提供的信息
3. **降低认知负担**：每款游戏 2-3 个标签的设计，让用户无需阅读大量描述即可了解游戏特色

## 四、已知待改进项

以下问题已在本次审核中识别，但暂不处理（除非用户有明确需求）：

| 问题 | 影响程度 | 说明 |
|------|----------|------|

| 标签长度不一 | 低 | 卡片布局可接受，长标签会自动截断 |
| C池未纳入本次 LLM 采集 | 中 | C池游戏尚未统一采集 LLM mechanics 标签 |

## 五、后续建议

如需进一步提升标签质量，可考虑以下方向（按优先级排序）：

1. **补充 C 池标签采集**：为 C 池（避坑指南池）游戏补充 LLM mechanics 标签，完成三池全覆盖
2. **标签翻译清洗**：对 LLM 返回的原始标签做二次清洗，去除英文括号注释，统一为纯中文
3. **标签长度限制**：在 LLM prompt 中增加"标签不超过 15 字"的要求，减少过长标签对布局的影响
