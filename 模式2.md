想要解决的问题：
我做了一个工具从steam上获取到最近3年上线的3000款回合制游戏。我希望从这些海量的游戏中找到高质量的回合制游戏（我正在构思做一款宝可梦like的游戏），我要怎么弄这些海量数据中发现那些有价值的游戏？

解决思路：
从3000款游戏中筛选出对你开发“宝可梦like（怪物收集/养成+回合制）”有价值的参考对象，是一个典型的数据清洗与分析过程。既然你已经有能力写工具抓取数据，建议你利用代码或Excel/数据库来进行高效的漏斗式筛选。
以下是为你量身定制的5步数据分析与游戏提炼指南：
第一步：数据清洗（去掉“工业垃圾”与无关项）
Steam上每年有大量粗制滥造的游戏，第一步必须大刀阔斧地砍掉无用数据，把3000款缩减到500款以内。
按评价数量过滤（市场验证筛选）：
排除评论数 < 50 的游戏：这些游戏基本没有经过市场检验，参考价值极低（除非是你认识的独具特色的小众游戏）。
建议保留区间：评论数在 200 - 5000 之间的游戏最适合独立开发者研究（这是“优秀独立游戏”的常客区间）。
按好评率过滤（质量筛选）：
排除好评率 < 70%（多半好评以下）的游戏：除非它带有强烈的“宝可梦”标签（这种可以留作“避坑指南”研究）。
按标签剔除（排除无关子品类）：
回合制是一个大类。你需要排除掉与“宝可梦like”相差甚远的子类。
黑名单标签（剔除）：棋盘游戏 (Board Game)、4X、大战略 (Grand Strategy)、成人/色情 (NSFW/Hentai)、文字冒险 (Text-Based)。
第二步：精准定位（寻找“宝可梦like”的核心锚点）
现在游戏库应该小很多了。接下来要通过Steam标签（Tags）的交集，找出与你构思最接近的游戏。
给每款游戏打分排序（权重系统），如果游戏包含以下标签，就增加它的权重分：
核心标签（权重最高）：生物收集 (Creature Collector)、怪物捕捉 (Monster Catching)、养成。
次级标签（高相关度）：JRPG、队伍导向 (Party-Based RPG)、战术角色扮演 (Tactical RPG)。
现代融合标签（寻找创新点）：Roguelite、牌组构建 (Deckbuilding)、开放世界。
筛选结果分类：
把经过这轮筛选出来的游戏分为两个池子：
A池（精准对标）：同时具备 回合制 + 生物收集 的高分游戏（如《磁带妖怪 Cassette Beasts》、《怪物圣所 Monster Sanctuary》、《Temtem》等）。
B池（机制参考）：具备优秀 回合制 机制，但不一定是抓宠的游戏（如《暗黑地牢》、《星之海》、《神之天平》等）。
第三步：数据挖掘与指标分析（发现 Hidden Gems）
对于筛选出的几十到上百款游戏，不要只看好评率，要用更聪明的数据指标来分析：
引入“威尔逊得分（Wilson Score）”或类似算法：
单纯看好评率不准（10个人100%好评 vs 10000个人95%好评）。用威尔逊区间算法给游戏重新排个真实口碑榜。
分析价格与时长的关系：
抓取它们的价格，看看目前市场上这类游戏的定价区间是多少，玩家的预期心理价位在哪里。
研究“褒贬不一”但带有“生物收集”标签的游戏（非常重要！）：
不要忽视失败的同类游戏。 找几款近期发售、玩法是宝可梦like，但评价在 50%-60% 的游戏。去读它们的差评。你会立刻知道如今的玩家最讨厌什么（比如：跑图太慢、遇敌率太高、进化设计敷衍、UI操作繁琐等）。这能帮你避开大坑。
第四步：定性研究（深度拆解Top 30）
数据工作结束后，你需要挑选出 20-30 款最具代表性的游戏进行人工拆解。
不用每一款都买，按照以下流程：
看评论区（提炼玩家G点与痛点）：
写个小爬虫或者用ChatGPT总结这些游戏的Steam高赞评论。玩家都在夸什么？（是怪物设计好看？还是属性克制好玩？还是剧情优秀？）
看实机视频（B站/YouTube搜 Gameplay）：
重点观察前20分钟的流程，以及后期的战斗画面。
观察重点： 回合制战斗的节奏感（打击感、动画速度能不能快进）、UI交互（选技能是不是要点很多次鼠标）。
亲自玩Top 5-10：
挑出最符合你胃口的几款，深度游玩。
第五步：为你的“宝可梦like”提取设计灵感
在研究这些优秀的回合制游戏时，带着你的项目目的去寻找答案：
痛点改良：传统的宝可梦游戏节奏偏慢、需要背属性克制表。近3年的新游戏是怎么优化的？（比如UI上直接提示效果拔群？引入了战棋走位？还是结合了肉鸽元素？）
数值与养成：同类独立游戏是怎么处理“肝度”的？有没有一键扫荡？经验共享是怎么设计的？
差异化卖点：现在做纯粹的“打道馆+抓宠”已经很难卷赢了。你需要看看最近火的同类游戏叠加了什么玩法（比如《幻兽帕鲁》叠加了生存建造，《磁带妖怪》叠加了形态融合，《怪物圣所》叠加了银河恶魔城）。

核心思路：先打品类标签，再按品类做不同的质量过滤
你应该把筛选流程做成一个**“分叉路口”**，具体分为以下三步来实操：
第一步：全量数据“查户口”（寻找宝可梦like基因）
在3000条原始数据中，先不要做任何好评率的删减。直接扫描每一款游戏的 Steam标签(Tags)。
新增一个数据列（比如叫 Is_Pokemon_Like），用代码或Excel的包含函数，判断它是否属于核心竞品：
规则：如果标签包含 生物收集 (Creature Collector) 或 怪物捕捉 (Monster Catching)。
动作：将 Is_Pokemon_Like 标记为 True（或 1），其余的为 False（或 0）。
第二步：分发到三个不同的“蓄水池”
现在，你的数据被分成了两派（宝可梦派 vs 普通回合制派）。接下来对这两派使用不同的好评率过滤标准：
蓄水池 A（神作参考池）：
条件：Is_Pokemon_Like == False 且 好评率 >= 75% 且 评论数 > 200
目的：寻找《星之海》、《暗黑地牢》等优秀游戏，学习它们的UI、战斗表现力、底层数值循环。低于75%的普通回合制直接丢弃（因为不值得学）。
蓄水池 B（核心竞品池 - 成功案例）：
条件：Is_Pokemon_Like == True 且 好评率 >= 75% 且 评论数 > 50
目的：这就是你最直接的竞争对手和学习对象（如《磁带妖怪》）。拆解它们为什么能成。
蓄水池 C（避坑指南池 - 失败/争议案例）：👇你要的答案在这里
条件：Is_Pokemon_Like == True 且 好评率在 40% - 74% 之间 且 评论数 > 50
目的：精准定位那些“有宝可梦核心玩法，但玩家不买账”的游戏。
第三步：如何研究蓄水池C（避坑指南池）？
筛选出蓄水池C后，你可能会得到 20-50 款游戏。不要去玩它们（浪费时间），而是用爬虫或手动去看它们的“差评集中点”。
