### 1.2 权重标签系统

**文档要求**：给游戏打标签权重分
- **核心标签**（最高权重）：生物收集、怪物捕捉、养成
- **次级标签**（高相关度）：JRPG、队伍导向、战术角色扮演
- **现代融合标签**（创新点）：Roguelite、牌组构建、开放世界

**任务清单**：

#### 1.2.1 定义标签权重常量

**注意**：标签分类需互斥，避免重复计数
- 核心标签和次级标签不重叠
- 现代融合标签独立统计

```typescript
// 标签分类定义（互斥）
const CORE_TAGS = [
  "Creature Collector",
  "Monster Catching",
  "Monster Taming",
  "Creature Collection",
  "养宠", "养成", "宠物养成", "怪物养成"  // 中文养成分支
];

const SECONDARY_TAGS = [
  "JRPG",
  "Party-Based RPG",
  "Tactical RPG",
];

// 现代融合标签（差异化卖点）
const MODERN_TAGS = [
  "Roguelite",
  "Roguelike",
  "Deckbuilding",
  "开放世界",
  "Open World",
  "Metroidvania",
  "银河恶魔城",
  "生存建造",
  "Survival",
  "Crafting",
  "牌组构建",
  "形态融合",  // Cassette Beasts特色
];
```

#### 1.2.2 计算权重得分

在 `transformGame` 函数中添加：
- `coreTagCount`: 核心标签匹配数量
- `secondaryTagCount`: 次级标签匹配数量
- `modernTagCount`: 现代融合标签匹配数量
- `tagWeight`: 综合权重分 (core*3 + secondary*2 + modern*1)

**计算逻辑**：
1. 先判断是否为核心标签，匹配则不再匹配次级
2. 次级标签只在非核心时计算
3. 现代标签独立计算，不受核心/次级影响

#### 1.2.3 前端展示

在游戏卡片上高亮显示匹配到的标签类型：
- 核心标签用 **金色标签**（如：👑 宝可梦Like）
- 次级标签用 **蓝色标签**
- 现代融合标签用 **紫色标签**（如：🎲 肉鸽融合）

#### 1.2.4 筛选功能增强

添加按标签类型筛选：
- 只看核心竞品（核心标签>0）
- 只看创新融合（现代标签>0）
- 综合排序（按权重分）

## 八、风险评估

1. **标签识别准确性**：部分游戏标签混乱，需要持续优化
   - **缓解**：标签分类使用模糊匹配，并提供人工调整接口

2. **价格数据完整性**：部分游戏价格数据可能缺失或为捆绑包价格
   - **缓解**：价格统计时过滤异常值（如$0或>$200）

3. **API性能**：增加计算量可能导致响应变慢
   - **缓解**：标签计算在数据加载时完成，结果缓存5分钟

4. **C池差评预测准确性**：基于标签的差评方向预测是简化方案
   - **缓解**：明确标注为"预测方向"，用户仍需去Steam验证
